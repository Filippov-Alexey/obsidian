# Second Brain — REST API Reference

Базовый URL: `http://127.0.0.1:5000`
Все endpoints отдают/принимают JSON (кроме import/export, где multipart или скачивание файла).
CORS открыт для всех источников (для локальной разработки).

## Health / Config

| Method | Path              | Description                                             |
|-------:|-------------------|---------------------------------------------------------|
| GET    | `/api`            | Карта всех endpoints (этот документ в JSON)             |
| GET    | `/api/health`     | Проверка соединения с БД                                |
| GET    | `/api/config`     | Получить тему + хоткеи + настройки приложения           |
| PATCH  | `/api/config`     | Обновить тему / хоткеи — persists в `config.json`       |

## Nodes (папки + заметки)

| Method | Path                           | Description                                                |
|-------:|--------------------------------|------------------------------------------------------------|
| GET    | `/api/nodes`                   | Полное дерево вложенных папок и заметок (рекурсивно)        |
| POST   | `/api/nodes`                   | Создать папку или заметку. Body: `{parent_id, title, is_folder}` |
| GET    | `/api/flat`                    | Плоский список всех узлов + всех связей (для графа, подсказок) |
| GET    | `/api/nodes/<id>`              | Узел + `backlinks` (список заметок, ссылающихся на него)    |
| PUT    | `/api/nodes/<id>`              | Обновить: `title` (переименование с правкой всех ссылок), `parent_id` (DnD), `content` (Markdown) |
| DELETE | `/api/nodes/<id>`              | Удалить (с каскадом вложенных и записей в links/tabs_session) |
| POST   | `/api/nodes/<id>/move`         | Перенести в другую папку: `{parent_id}` (null = корень)     |
| POST   | `/api/nodes/<id>/duplicate`    | Создать копию заметки/папки с суффиксом `(copy N)`          |
| GET    | `/api/nodes/<id>/export`       | Скачать заметку как отдельный `.md` файл                    |
| GET    | `/api/nodes/<id>/outlinks`     | Исходящие wiki-ссылки заметки (распарсенные из `[[...]]`)   |
| GET    | `/api/nodes/<id>/backlinks`    | Только обратные ссылки (список)                             |

## Links / validation

| Method | Path                   | Description                                                |
|-------:|------------------------|------------------------------------------------------------|
| GET    | `/api/broken-links`    | Список заметок с `[[ссылками]]` на несуществующие заметки   |

## Tags

| Method | Path                 | Description                                                |
|-------:|----------------------|------------------------------------------------------------|
| GET    | `/api/tags`          | Все теги с количеством заметок: `[{tag, count}, ...]`       |
| GET    | `/api/tags/<tag>`    | Заметки, содержащие `#tag`                                  |

## Search / Lists

| Method | Path                                    | Description                                               |
|-------:|-----------------------------------------|-----------------------------------------------------------|
| GET    | `/api/search?q=&scope=&limit=&is_folder`| Поиск по `title`, `content` или `all`. `scope=title\|content\|all` |
| GET    | `/api/recent?limit=10`                  | Недавно изменённые заметки (по `updated_at DESC`)         |
| GET    | `/api/stats`                             | Сводка: кол-во заметок, папок, ссылок, объём КБ, теги    |

## Graph

| Method | Path                           | Description                                             |
|-------:|--------------------------------|---------------------------------------------------------|
| GET    | `/api/graph`                   | Узлы + связи для графа. `?include_id=<id>` — эго-граф 1-hop |

## Tabs / Session

| Method | Path                         | Description                                                  |
|-------:|------------------------------|--------------------------------------------------------------|
| GET    | `/api/session`               | Получить вкладки (массив `{node_id, position_order, is_active}`) |
| POST   | `/api/session`               | Перезаписать все вкладки сразу: `{tabs: [...]}`              |
| POST   | `/api/tabs`                  | Открыть одну вкладку: `{node_id}` (становится активной)       |
| DELETE | `/api/tabs/<node_id>`        | Закрыть вкладку                                              |
| PUT    | `/api/tabs/<node_id>/activate` | Сделать вкладку активной                                     |

## Import / Export

| Method | Path                        | Description                                                   |
|-------:|-----------------------------|---------------------------------------------------------------|
| GET    | `/api/export`               | Скачать полный zip-архив: `.md` файлы + `meta.json` + `session.json` |
| POST   | `/api/import`               | Загрузить zip. `multipart/form-data`: `file` (zip), `mode=merge\|replace` |
| POST   | `/api/import/markdown`      | Загрузить одну заметку: `multipart/form-data`: `file` (.md), опционально `parent_id`, `title` |

## Коды ответов

- `200 OK` — успех
- `201 Created` — ресурс создан
- `400 Bad Request` — неверные параметры / пустой title
- `404 Not Found` — узел не существует
- `409 Conflict` — дубликат имени или ошибка целостности
- `500 Internal Server Error` — ошибка БД / чтения конфига

Ошибки возвращаются как `{"error": "message"}`.

## Поведенческие особенности

- **Связи** при сохранении контента (PUT с `content`) автоматически
  пересчитываются регулярным выражением `[[Название]]` / `[[Название|алиас]]`.
- **Переименование** заметки (PUT `title`) автоматически заменяет
  все `[[старое]]` на `[[новое]]` во всех остальных заметках и
  пересчитывает таблицу `links`.
- **Удаление** узла каскадно удаляет его потомков, все их связи и
  записи в `tabs_session` (FK `ON DELETE CASCADE` + явная очистка).
- **Drag-and-Drop** перенос: при валидации запрещено помещать папку в
  свою подпапку (защита от циклов).
- **Импорт zip** без `meta.json` восстанавливает иерархию по каталогам
  архива и автоматически выстраивает связи по `[[...]]` в markdown.
- **Импорт zip с `meta.json`** сохраняет внутренние id-маппинг и связи
  между импортированными заметками.
