# Second Brain — локальный аналог Obsidian (Flask + MySQL + Vanilla JS)

## Структура проекта

```
├── app.py                 # Flask-бэкенд (REST API)
├── config.json            # Конфигурация (БД, хост/порт, тема, хоткеи)
├── schema.sql             # MySQL-схема (опционально, app.py создаёт таблицы сам)
├── requirements.txt
├── templates/
│   └── index.html         # Шаблон основной страницы
└── static/
    ├── css/style.css      # Все стили (тёмная/светлая темы)
    └── js/app.js          # Весь фронтенд на чистом JS (без сторонних библиотек)
```

## Установка и запуск

1. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```

2. Настройте параметры подключения к MySQL в `config.json`:
   ```json
   { "database": { "host": "127.0.0.1", "port": 3306,
                   "user": "root", "password": "",
                   "database": "secondbrain" } }
   ```

3. (Опционально) Создайте базу вручную:
   ```bash
   mysql -u root -p < schema.sql
   ```
   Если этого не сделать, `app.py` при старте автоматически создаст все таблицы
   (но базу `secondbrain` нужно создать вручную, или подставить в
   `config.json` имя уже существующей).

4. Запуск:
   ```bash
   python app.py
   ```

5. Откройте в браузере: <http://127.0.0.1:5000>

## Архитектура БД (MySQL)

- `nodes` — папки и заметки (self-referencing через `parent_id`; флаг `is_folder`).
- `links` — wiki-связи между заметками (`source_id → target_id`),
  пересчитываются автоматически при каждом сохранении контента.
- `tabs_session` — состояние вкладок рабочего пространства (для
  восстановления после перезагрузки).

## REST API (JSON)

| Метод | Путь                  | Назначение                                          |
|------:|-----------------------|-----------------------------------------------------|
| GET   | `/api/nodes`          | Дерево папок/заметок (рекурсивно)                   |
| POST  | `/api/nodes`          | Создать папку или заметку                           |
| GET   | `/api/nodes/<id>`     | Данные узла + обратные ссылки                       |
| PUT   | `/api/nodes/<id>`     | Переименование / перенос / сохранение контента      |
| DELETE| `/api/nodes/<id>`     | Удаление (с каскадом)                               |
| GET   | `/api/flat`           | Плоский список всех узлов + всех связей (для графа) |
| GET   | `/api/session`        | Состояние вкладок                                   |
| POST  | `/api/session`        | Сохранить состояние вкладок                         |
| GET   | `/api/config`         | Тема + хоткеи для фронтенда                         |

## Возможности фронтенда

- Файловый менеджер (дерево с папками/заметками), Drag-and-Drop.
- Контекстное меню (новая заметка/папка, переименование, удалить).
- Вкладки с автосохранением сессии в БД.
- Редактор Markdown с тремя режимами: **Source**, **Live Preview**, **Preview**.
- Wiki-ссылки `[[Название]]` — клик открывает заметку, битая ссылка предлагает создать.
- Автодополнение: `[[` — список заметок, `#` — список тегов.
- Граф связей на `<canvas>` 2D (force-directed layout):
  - Кулоновское отталкивание + пружинки + трение.
  - Zoom (wheel), pan (drag), drag узлов, hover-подсветка.
  - Двойной клик по узлу — открыть заметку.
- Горячие клавиши настраиваются в `config.json` (применяются после перезапуска).
- Тёмная/светлая тема.
- Переименование заметки автоматически обновляет все `[[старое]]` на
  `[[новое]]` во всех других заметках.

## Критерии приёмки

- `python app.py` поднимает сервер, таблицы создаются автоматически.
- После F5 восстанавливаются вкладки, активная вкладка, контент.
- Циклические ссылки (A↔B) не ломают бэкенд и корректно рисуются на графе.
- Во фронтенде нет внешних `<script src="...">` на сторонние CDN.
